Technology

Compute where the data lives

Most of the energy in edge AI is spent moving data, not computing on it. Our IP does the work inside the memory, so always-on inference runs on a fraction of the power.

The memory wall

The bottleneck is data movement

A conventional NPU streams weights from memory for every inference. On always-on edge models, that traffic, not the maths, is what drains the battery.

Ximplic in-memory

Compute in SRAM
✓Weights stay resident inside the compute array
✓Inference runs in place, with no data shuttling
✓Energy scales with useful work, not memory traffic
✓Standard SRAM, buildable today, sized to the target budget

Conventional NPU

von Neumann
✕Weights streamed from memory every inference

✕Battery drains fast on small, frequently-run models
How it works

The memory is the compute engine

The SRAM array does the multiply-accumulate itself. Weights are written once; activations stream in, the array computes, and only the result comes out.

  • Weights resident on-chip, written once
  • Compute happens in the array, not beside it
  • INT8 to INT1 precision, configurable array size
  • Standard-cell digital design, portable across foundries
Activations stream in, the resident weights compute in place, only the result comes out.
The energy difference

Where the energy goes

The compute cost is similar either way. The difference is the data movement around it, which in-memory computing largely removes.
