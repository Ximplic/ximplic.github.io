Compute where the data lives

Most of the energy in edge AI is spent moving data, not computing on it. Our IP does the work inside the memory, so always-on inference runs on a fraction of the power.

01 / The memory wallWhy efficiency is hard

The bottleneck is data movement

A conventional NPU stores weights in memory and streams them to the compute units for every inference. On the small models that run all the time at the edge, that back-and-forth traffic, not the maths, is what drains the battery.

02 / How it worksIn-memory compute

The memory is the compute engine

Instead of a separate memory and a separate processor, the SRAM array itself performs the multiply-accumulate. Weights are written once and stay put; activations stream in, the array computes, and only the result comes out.

  • Weights resident on-chip, written once
  • Compute happens in the array, not beside it
  • INT8 to INT1 precision, configurable array size
  • Standard-cell digital design, portable across foundries
The accelerator drops into a host SoC over AXI; the compute array sits inside it.
