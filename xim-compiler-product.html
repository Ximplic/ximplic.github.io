01 / What it isXIM Compiler

From a trained model to running on the accelerator

The toolchain that turns a standard ONNX model into a hardware-ready deployment. It quantizes, calibrates, prunes, schedules across the compute banks, and emits the descriptor the hardware runs.

A model-agnostic core. A thin frontend reads the ONNX graph; the core makes the hardware decisions: tiling wide layers, splitting columns, allocating banks, and choosing per-layer precision. Convolutional, depthwise-separable and dense networks all use the same path.

Quantization is where it earns its keep. Per-layer mixed precision keeps sensitive layers at INT8 while the rest run INT4, with structured pruning and a requantization path calibrated so software matches silicon, confirmed on the board.

Deploy a model in a few lines

A clean Python API takes an ONNX model and calibration data to a hardware deployment:

# normalize an ONNX graph into the compiler IR layers = onnx_frontend("model.onnx", calibration) # quantize, prune, schedule -> hardware descriptor model = compile_ir(layers) # emit the deploy header the accelerator runs emit_model_header(model)
02 / CapabilitiesProduction toolchain

A deployment toolchain, not a demo script

Standard ONNX frontend

Bring a model from any framework, exported to ONNX. No proprietary model format to learn.

Multi-precision quantization

INT8 and INT4 with per-layer mixed precision, calibrated to the hardware datapath so software and silicon agree.

Structured pruning

Hardware-aware column pruning the accelerator can actually skip, with prune-aware fine-tuning to hold accuracy.

Tiling & scheduling

Wide layers are tiled and scheduled across the compute banks automatically. Large models map to fixed silicon without hand-tuning.

Hardware-faithful

A fail-loud contract rejects anything the hardware can't run, and the deployment is checked bit-for-bit against the chip.

Multiple model families

Keyword-spotting CNNs, depthwise-separable vision nets and dense autoencoders deploy through the same core.
